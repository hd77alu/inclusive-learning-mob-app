Inclusive Learning Platform — Flutter App: Current State

Project: Inclusive Mobile Learning and Skills Platform for Persons with Disabilities in Rwanda
Stack: Flutter (Dart), Firebase Auth, Firestore, BLoC, SharedPreferences

---

WHAT THE APP IS:
A mobile app that helps persons with disabilities in Rwanda access learning, connect with mentors,
and manage their skills. It uses Firebase for authentication and data storage, BLoC for state
management, and SharedPreferences for local settings.

---

SCREENS BUILT:

1. App Entry / Home Screen (app_outlook_screen.dart)
   - Teal header with app title "Inclusive Learning Platform"
   - Hero illustration with diverse person avatars
   - Icon grid: Lessons, Videos, Creative, Movies, Chat, Forum (show "coming soon" snackbar)
   - "Start here!" button navigates to Sign Up
   - Profile icon opens a bottom sheet showing guest user info
   - Back button support

2. Sign Up Screen (sign_up_screen.dart)
   - Firebase Authentication sign-up flow
   - Form with email and password fields
   - Navigates to the main app after successful sign-up

3. Course Completion Screen (course_completion_screen.dart)  ← YOUR SCREEN
   - Auth guard: redirects to /signup if user is not logged in
   - Dark green/black gradient background
   - Entrance animation: trophy icon scales in with elastic curve, badge and text fade in
   - Teal trophy icon with glow shadow effect
   - "Congratulations!" teal rounded badge (animated)
   - Certificate card showing authenticated user's email from Firebase Auth
   - "Proceed to Mentorship Hub" full-width teal button at the bottom
   - Responsive: portrait layout stacks vertically, landscape layout splits into two columns
   - Wrapped in SingleChildScrollView + ConstrainedBox — no overflow on any screen size

4. Mentorship Hub Screen (mentorship_hub_screen.dart)  ← YOUR SCREEN
   - Auth guard: redirects to /signup if user is not logged in
   - Teal header with subtitle "Inclusive Learning Platform" + title "Mentorship Hub"
   - Back arrow and profile icon in header
   - Rounded search bar with clear (X) button when text is entered
   - Filter chips: All, Sign Language, Braille — live filtering by name, role, and tags
   - Mentor cards loaded from Firestore via BLoC (MentorshipBloc)
   - Responsive: single column list on phones, 2-column grid on tablets/landscape (width >= 600px)
   - Staggered card entrance animation: each card slides up and fades in with 80ms delay per index
   - Each mentor card includes:
       - Teal avatar banner at the top
       - "ONLINE NOW" green badge (when isOnline: true in Firestore)
       - Mentor name (overflow-safe) + star rating
       - Role/title subtitle
       - Description (2-line truncated)
       - Tag chips (e.g. Sign Language, Job Coaching)
       - 3 action buttons: Save/Bookmark, Call, Video
   - Save/Bookmark button:
       - Connected to Firestore via BLoC (ToggleBookmark event)
       - Optimistic UI update — bookmark state changes instantly without waiting for Firestore
       - Reverts automatically if Firestore write fails
       - Shows SnackBar: "Mentor saved!" or "Bookmark removed"
       - Animated container changes color when bookmarked
   - Call and Video buttons show confirmation AlertDialog with mentor name
       - On confirm: shows SnackBar "[action] request sent!"
   - CircularProgressIndicator (teal) while loading
   - SnackBar shown if mentors fail to load
   - Empty state: icon + "No mentors available yet" + "Check back soon!"
   - Tapping a card navigates to Mentor Profile screen

5. Mentor Profile Screen (mentor_profile_screen.dart)  ← YOUR SCREEN
   - SliverAppBar with teal expandable header and mentor avatar
   - Online indicator dot on avatar (green dot when isOnline: true)
   - Mentor name, role, star rating
   - Tag chips
   - "About" section with full description
   - Stats row: Sessions (120+), Students (85+), Years Exp. (5+)
   - "Book a Session" button opens a dialog with:
       - Time slot selection chips: Morning, Afternoon, Evening
       - Optional note text field
       - Cancel and Confirm Booking buttons
       - On confirm: SnackBar "Session booked with [name] — [time slot]" (green)
   - "Send Message" outlined button with SnackBar feedback
   - Fully scrollable with CustomScrollView + SliverToBoxAdapter

6. Profile Screen (profile_screen.dart)
   - Shows authenticated user's email from Firebase Auth
   - Logout button in AppBar and at bottom of screen
   - Logout calls FirebaseAuth.instance.signOut() then redirects to /signup
   - Preferences tile navigates to Preferences screen
   - My Skills tile navigates to /skills

7. Preferences Screen (preferences_screen.dart)
   - 3 settings saved locally using SharedPreferences:
       - Dark Mode (ON/OFF)
       - Notifications (ON/OFF)
       - Language: English / French / Kinyarwanda
   - Settings persist across app sessions

8. My Skills Screen (my_skills_screen.dart)
   - Full CRUD connected to Firestore:
       - Add skill via dialog (name + level)
       - View all skills for the logged-in user
       - Edit existing skill
       - Delete skill
   - Skills stored at: users/{userId}/skills/{skillId}
   - SkillsBloc handles all business logic
   - SnackBars for: added, updated, deleted, and error feedback
   - CircularProgressIndicator while loading

FIRESTORE STRUCTURE:

  mentors/
    {mentorId}/
      name        (string)
      role        (string)
      rating      (number)
      description (string)
      tags        (array of strings)
      isOnline    (boolean)

  users/
    {userId}/
      bookmarks/
        {mentorId}/
          mentorId  (string)
          savedAt   (string - ISO date)
      skills/
        {skillId}/
          name   (string)
          level  (string)
